Requisitos No Funcionales

Según Metaute (2016), los RNF están relacionados con características externas de calidad como seguridad, facilidad de uso, interfaz práctica, velocidad de procesamiento y amigabilidad.

Facilidad de uso

ID. Requisito Descripción
RNF-001 Publicidad, antes de intensificar el uso del sistema los usuarios deben conocer su modo de uso.
RNF-002 Actualizar la información por ingreso o retiro de algún trabajador.
RNF-003 Diseño intuitivo, interfaz sencilla y carga de información rápida.

Confiabilidad

ID. Requisito Descripción
RNF-004 El sistema debe estar disponible las 24 horas del día.
RNF-005 Debe asegurar la permanente actualización de la base de datos.

Ambiente de trabajo (Performance)

ID. Requisito Descripción
RNF-006 Tiempo de respuesta promedio de 15 segundos. La base de datos debe soportar grandes cantidades de información. Configuración adecuada del equipo para instalación.

Restricciones de diseño

ID. Requisito Descripción
RNF-007 El sistema se desarrollará en Python y Android Studio.
RNF-008 Se requiere licenciamiento para el desarrollo del software.

Seguridad

ID. Requisito Descripción
RNF-009 Las contraseñas deben ser almacenadas encriptadas.
RNF-010 Realizar backups de los datos en disco o cinta.
RNF-011 Todas las operaciones de consulta y modificación deben ser auditadas.

Documentación y ayuda

ID. Requisito Descripción
RNF-012 Capacitación a los usuarios para un buen manejo del sistema.
RNF-013 Manuales de usuario.

Interfaz de Usuario

ID. Requisito Descripción
RNF-014 Será manejado en un dispositivo móvil Android.
RNF-015 Colores: gris oscuro, azul claro y blanco.
RNF-016 Texto negro, fuente Arial, tamaño 11.

Interfaces de comunicación

ID. Requisito Descripción
RNF-017 El acceso al software será vía APP Android, asegurando estabilidad y seguridad.
RNF-018 Comunicación con la interfaz de usuario.
