Modelo FURPS

Framework para la Especificación de Requisitos No Funcionales

¿Qué es FURPS?

FURPS es un acrónimo que representa un modelo para la clasificación de requisitos de software, desarrollado por Robert Grady en Hewlett-Packard. Este framework se utiliza para categorizar los requisitos no funcionales de un sistema, proporcionando una estructura sistemática para evaluar la calidad del software. FURPS significa Functionality (Funcionalidad), Usability (Usabilidad), Reliability (Confiabilidad), Performance (Rendimiento) y Supportability (Soporte).

F - Functionality

Funcionalidad

Se refiere a las características funcionales y capacidades del sistema, incluyendo el conjunto de funciones y la adecuación de estas funciones.

Ejemplos:

  • Autenticación y autorización de usuarios
  • Integración con APIs externas
  • Procesamiento de transacciones
  • Generación de reportes
  • Gestión de datos y almacenamiento

U - Usability

Usabilidad

Evalúa qué tan fácil es para los usuarios aprender, usar y obtener ayuda del sistema. Incluye aspectos de la experiencia del usuario y la interfaz.

Ejemplos:

  • Interfaz intuitiva y fácil de navegar
  • Tiempo de aprendizaje del sistema
  • Accesibilidad para usuarios con discapacidades
  • Ayuda contextual y documentación
  • Consistencia en el diseño de la interfaz

R - Reliability

Confiabilidad

Capacidad del sistema para mantener su nivel de rendimiento bajo condiciones establecidas durante un período determinado.

Ejemplos:

  • Disponibilidad del sistema (99.9% uptime)
  • Frecuencia de fallas y recuperación
  • Tolerancia a fallos y redundancia
  • Backup y recuperación de datos
  • Manejo de errores y excepciones

P - Performance

Rendimiento

Se refiere a la velocidad, eficiencia y capacidad de respuesta del sistema bajo diferentes cargas de trabajo.

Ejemplos:

  • Tiempo de respuesta (< 2 segundos)
  • Throughput (transacciones por segundo)
  • Uso de recursos (CPU, memoria)
  • Escalabilidad y capacidad de carga
  • Optimización de consultas de base de datos

S - Supportability

Soporte

Facilidad con la que el sistema puede ser mantenido, configurado, instalado y soportado por el equipo técnico.

Ejemplos:

  • Facilidad de instalación y configuración
  • Capacidad de monitoreo y logging
  • Documentación técnica completa
  • Modularidad y mantenibilidad del código
  • Compatibilidad con diferentes plataformas

Aplicación de FURPS en Nuestro Proyecto

FFuncionalidad

Detección de emociones caninas, análisis de comportamiento, generación de reportes

UUsabilidad

Interfaz intuitiva para propietarios, navegación simple, visualización clara de datos

RConfiabilidad

Disponibilidad 24/7, precisión en la detección, recuperación ante fallos

PRendimiento

Respuesta en tiempo real, procesamiento eficiente de datos de sensores

SSoporte

Actualizaciones automáticas, documentación completa, compatibilidad multiplataforma
