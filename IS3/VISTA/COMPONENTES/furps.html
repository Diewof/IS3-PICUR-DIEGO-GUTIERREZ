FURPS

El modelo FURPS es un marco de referencia utilizado en ingeniería de software para clasificar y evaluar los requisitos de calidad de un sistema. FURPS es un acrónimo que representa cinco categorías principales de atributos de calidad.

¿Qué significa FURPS?

F - Functionality (Funcionalidad)

Capacidades del sistema, características, seguridad y funciones generales.

U - Usability (Usabilidad)

Factores humanos, ayuda, documentación y facilidad de uso del sistema.

R - Reliability (Confiabilidad)

Frecuencia de fallos, capacidad de recuperación, predictibilidad y exactitud del sistema.

P - Performance (Rendimiento)

Velocidad de respuesta, rendimiento, uso de recursos, eficiencia y capacidad de carga.

S - Supportability (Mantenibilidad)

Testabilidad, adaptabilidad, mantenibilidad, compatibilidad, configurabilidad, facilidad de instalación y localización.

Importancia del Modelo FURPS

El modelo FURPS proporciona un enfoque sistemático para identificar, clasificar y evaluar los requisitos no funcionales de un sistema de software. Esto ayuda a los desarrolladores y stakeholders a tener una visión completa de la calidad esperada del producto final, facilitando la planificación, el diseño y las pruebas del sistema.
